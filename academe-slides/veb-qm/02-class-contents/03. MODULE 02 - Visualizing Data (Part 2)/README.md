
This is not a single lecture. This should be spread probably around 4 sessions. Read more to know why.

Explain to them that basic understanding of data and visualizing skills are needed to at least start understanding Data Analysis in python.
That is why in this lecture you are going to discuss visualizing data by using Microsoft Excel

The lesson should be easy and straightforward. The hard part is going to be the Excel. Although it is Lecture 01, you are absolutely going to granulate this topic 
this might take multiple lectures because you are going to pace yourself with the student's ability to follow you on the Excel part.

When you start getting in the Excel part. Have them download all the (4) datasets. 
Then afterwards have them look at the file without opening it yet, and ask them whether they notice anything different.
What I'm trying to achieve is for them to see that the file extension of the file they download is .csv and not .xlsx

After that explain to them why most datasets are in .csv and why is it useful to do share/get it as that rather than .xlsx
The main reason is the .csv is basically just a text -- theres no objects, no formatting, no complicated stuff -- but it is organized data in a sense that each data is separated by comma.
Any softwares can use the .csv basically they can read it as text and just parse it with the comma delimeter. Any professional would prefer being sent a .csv dataset rather than a .xlsx

Now you will have them follow your lecture in Excel. Before you start that always remind them to regularly SAVE THEIR WORK!!!

Have them import all those datasets into Excel. Theres basically (2) ways
	(A) have them import it manually. Either by copy-pasting it or by opening the .csv and then moving it one by one to the target .xlsx
		- The downside from this is that the imported dataset is not yet a "table", so teach them how to make a table a "table" in Excel (use Ctrl + T to make things easier).
		- Also explain to them that the data in Excel is now separate from the actual dataset that they downloaded, any changes they make in the original .csv won't take effect on the .xlsx
	(B) have them import it through Data -> Import Text/CSV.
		- The only downside to this is that some machine (outdated Excel) won't have Import CSV only Import Text. 
			- If they don't have the Import CSV option, just have them use the first (A) method.
		- The best thing about this is that it data stays connected to the .csv file through the "Power Queries & Connections" panel
			- which means any changes in data to the original will reflect to the Excel file after clicking the "Refresh" data

After this proceed with the lecture, then you come to a point in which you will be teaching them how to create a "Frequency Table"
	- Have them create a pivot table first (Insert -> Pivot Table), teach them how a pivot table works
		- Pivot table is basically a table that is referring to an actual original table without reorganizing the reference table.
		
Afterwards, you will be discussing Pivot Charts (charts that are making Pivot Table as the basis)
	You will be creating these graphs according to the dataset
		- Bar Graphs 
		- Pie Graphs
